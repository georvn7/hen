I'm generating synthetic training data to train an LLM-based AI coding agent for C++ debugging.

YOUR TASK
==============================================================================

OBJECTIVE:
Optimize the provided 'TRAJECTORY SEGMENT TO OPTIMIZE' by finding a more efficient sequence of actions (from first run_test to the next fix_function) that achieves the same result.

WHAT'S ALREADY DONE:
In the provided segment, consecutive fix_function steps (if any) targeting the same function have already been merged and marked with "NOTE: Merge step X into step Y" annotations.

OPTIMIZATION GOAL:
From first run_test to last fix_functin sequence, identify only the necessary steps that will produce the same effect on the source code and test outcome. Create a distilled sequence that:
- Starts with the first run_test
- Includes only essential information-gathering actions
- Contains the final consolidated fix_function
- Ends with the final fix_function
- Mimics expert-level debugging without intermediate fixes

ROLLOUT TARGET:
[[ideal_target_guidance]]

[[rare_action_guidance]]

CRITICAL: The optimized sequence must show LOGICAL, STEP-BY-STEP progression where each action is a natural consequence of the information available up to that point.

Think of it as: "An expert debugger works through the problem efficiently, where each step makes sense given only what they know so far."

This means:
- Each information-gathering action should be justified by what was discovered in previous steps
- You cannot request information about something you haven't identified yet
- The fix_function should only appear after enough information has been gathered to understand the problem
- Each step should answer: "Given what I know NOW, what's the most logical next action?"
  
The sequence should feel like watching an experienced developer work efficiently - not jumping to conclusions, but also not taking unnecessary detours.

HOW THE AI AGENT WORKS (FOR YOUR UNDERSTANDING)
